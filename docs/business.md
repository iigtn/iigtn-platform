# Business — クライアント案件への展開構想

> 本構成は個人ポートフォリオ用ですが、**ほぼ同じテンプレートが小〜中規模の案件に流用できる** と考えています。
> このドキュメントは **「将来の構想」** であり、現時点で実案件があるかは別問題です。
>
> ⚠️ 「○○%流用可能」「月額○○円で運用可能」といった数字は **目安** です。実際の案件では要件・規模・SLA で大きく変動します。

---

## 応用可能な案件タイプ（目安）

| 業種・規模 | 案件イメージ | 流用しやすさ |
|---|---|---|
| 士業（弁護士・税理士・社労士） | 事務所サイト + 問い合わせ + ブログ | 高 |
| 美容・整体・歯科クリニック | 院長ブログ + 症例紹介 | 高（予約は別 SaaS 連携） |
| 飲食・小売（個店） | メニュー + ブログ + 来店導線 | 高 |
| BtoB 中小企業（10〜50名） | コーポレート + 採用 + IR 簡易 | 中（採用要件次第） |
| スタートアップ LP | プロダクト紹介 + 資料請求 | 中（LP テスト多発時は別構成検討） |
| 自治体 / NPO | 情報発信 + 申請フォーム | 中（個人情報・帳票要件次第） |

> ⚠️ 流用率の数字（v3 まで「80〜90%」と記載）は「読み手に印象を与えすぎる」と判断したため、**「高 / 中」のレンジ表記** に変更しています。実態は要件次第です。

---

## 流用しにくい / 別構成が必要なケース

逆に、本構成では **適合しない / 重装になる** ケースも明記します。営業判断にも使える情報。

| 状況 | 理由 |
|---|---|
| 会員登録・ログイン必須 | Cognito / Auth0 など認証層を追加。複雑度が一段上がる |
| 大量画像 / 動画配信が中核 | CloudFront + S3 のままでも動くが、画像最適化と転送量設計が別案件級 |
| EC（決済） | Stripe / Shopify 連携前提。本構成のフォーム + Lambda では不足 |
| 高頻度更新の DB アクセス | DynamoDB のデータモデルで詰むことがある。RDS + Aurora Serverless v2 を検討 |
| SLA 99.95%+ | マルチリージョン / RPO/RTO 設計が必要。月額が一段上がる |
| 法定保存（医療・金融） | KMS CMK / 監査ログ / コンプラ要件で構成見直し |

## クライアントへの提案価値（条件付き）

| 提案軸 | 訴求内容 | 条件・前提 |
|---|---|---|
| **コスト** | **月額 1,000〜3,000 円台** が目安 | PV 数千〜数万、画像中心でない、SES 月数百通以下 |
| **可用性** | CloudFront SLA 99.9% | 単一リージョン構成。リージョン障害時は影響あり |
| **セキュリティ** | OS / ミドルウェアの脆弱性管理が不要 | ⚠️ Lambda ランタイム・Node・npm 依存・OSS の脆弱性は別途管理が必要（SCA / SAST） |
| **スケール** | バズ・キャンペーン時に自動スケール | DynamoDB On-demand と SES quota は事前準備が必要 |
| **保守工数** | OS パッチ運用が不要 | **CMS 更新 / 画像差し替えなど運用業務は別** |
| **ベンダーロック対策** | Terraform で構成全公開、引き継ぎ容易 | AWS ロックインは残る（マルチクラウド前提なら設計から見直し） |

## 適用イメージ（モデル提案）

```
クライアント： 従業員 20 名の士業事務所
要件： コーポレート + ブログ + 問い合わせ + 採用
予算： 初期 30〜50 万円、月次運用 1〜3 万円

提案内容：
  - 本テンプレートを clone してドメイン・配色・コンテンツ差し替え
  - microCMS を導入（事務員がブラウザから記事更新可）
  - GA4 / Search Console / Microsoft Clarity 設置
  - 月次レポート（PV / 問い合わせ件数 / コスト）を自動生成

提供形態：
  - 構築一括 + 月次保守契約
  - 1 年後にソース・Terraform State を一式引き渡しオプション

⚠️ 月額 1〜3 万円には以下が含まれる前提：
  - AWS 課金実費の上乗せ
  - 監視・障害対応・記事更新代行・軽微な修正
  - SLA は best-effort（24h 以内対応など）
含まれないもの：
  - 大規模リニューアル・新機能開発
  - SES ドメイン認証・DNS 設定変更などの初期作業
```

## パッケージ化の方向性（構想）

将来的な可能性：

- **テンプレート販売**: GitHub Sponsors / Gumroad
- **保守サービス**: 月額固定での監視・記事更新代行
- **同業向けトレーニング**: 「インフラエンジニアのためのサーバレス案件獲得」

> ⚠️ これらは **構想段階** です。実際の案件実績ができたら、匿名化した事例を `case-studies/` に追加していきます。
> 現時点では「この設計を案件に転用したい」という意思の表明であり、実績ベースではありません。

## 営業時に正直に言うべきこと

> クライアント関係を長期化させるために、**最初に言いにくいことを言っておく** のが結局得です。

- **WordPress の方が向いているケースは存在する** — 編集者が多数 / プラグインで完結する要件 / 既存のレンサバ運用に慣れている、などは無理にサーバレス化しない方が良い
- **AWS ロックインは残る** — 「ベンダーロックなし」と書いたが、Terraform を移植可能にしているだけで AWS のサービス自体は残る
- **障害時のリスクは負う必要がある** — 個人運用なら自己責任、業務委託なら賠償条項を契約に明記
- **インフラ料金とは別に保守費用は発生する** — 「AWS が月 1,000 円」と「保守として月 2 万円」は別の話
