## 月額コスト試算（無風時 / 月間 PV 5,000 想定）

| サービス | 想定使用量 | 月額（円・概算） |
|---|---|---:|
| Route 53 Hosted Zone | 1 zone | ~75 |
| Route 53 Queries | ~50,000 query | ~5 |
| ACM | パブリック証明書 | 0（CloudFront 用は無料） |
| S3 Storage | 1 GB | ~3 |
| S3 Requests | GET 50,000 / PUT 数百 | ~3 |
| CloudFront 転送量 | 5 GB | ~85 |
| CloudFront Requests | 50,000 | ~10 |
| API Gateway HTTP API | 1,000 requests | ~0.2 |
| Lambda 実行 | 1,000 invocations / 128MB / 200ms | 無料枠で吸収 |
| **Lambda 副次（CloudWatch Logs 取り込み）** | ~100MB | ~10 |
| DynamoDB On-demand | Write 1,000 / Read 5,000 | ~1 |
| **SES** | 1,000 通送信（ドメイン認証済） | ~15（送信数で変動） |
| CloudWatch Logs（API GW + Lambda 合計） | 1 GB 取り込み・30 日保管 | ~80 |
| CloudWatch Alarms | 5 個 | ~75 |
| **合計（概算）** |  | **約 360〜420 円/月** |

> ⚠️ **「ほぼ無料」と書ける理由**：Lambda・API Gateway は無料枠と少額課金で月数十円に収まるが、**CloudWatch Logs と Alarm が現実のベース費用** になる。これを見落とすと「思ったより高い」事故が起きる。

## トラフィック増加時のコスト変動

| シナリオ | 月間 PV | 月間データ転送 | 推定月額 | 主なボトルネック |
|---|---:|---:|---:|---|
| 平常時 | 5,000 | 5 GB | ~400 円 | — |
| ブログがバズ | 50,000 | 50 GB | ~1,500 円 | CloudFront 転送量 |
| 中規模個人サイト | 200,000 | 200 GB | ~5,500 円 | CloudFront + Logs |
| Bot 攻撃を受ける | — | — | **数千円〜数万円** | CloudFront 転送量、Lambda 同時実行、API GW |

> 攻撃想定は「想定外の」コストではなく、**運用前から織り込むべき** リスク。AWS Budgets の閾値アラートで早期検知する。

## コスト最適化

| 項目 | 対策 |
|---|---|
| **CloudFront 転送量** | キャッシュ TTL を長め（24h〜）に設定、画像は WebP / AVIF 化 |
| **CloudFront Invalidation** | **paths 単位で月 1,000 paths まで無料**、超過分は **path あたり約 $0.005**（≒ 0.75 円）。`/*` は wildcard 1 path 扱いだが、リリース毎に大量 paths を投げる workflow にしないこと |
| **CloudWatch Logs** | Retention を **14〜30 日** に明示設定（デフォルト無期限の事故を防ぐ） |
| **Lambda** | arm64 (Graviton) で約 20% 安。bundle サイズ削減でコールドスタートも短縮 |
| **DynamoDB** | アクセスが少ない間は On-demand、定常負荷が読めたら Provisioned + Auto Scaling |
| **WAF** | 平常時は導入せず、攻撃検知してから有効化（マネージドルール 1 つで月 1,000 円〜） |
| **Budgets** | 月 1,000 円・3,000 円・10,000 円の段階アラート（無料） |
| **Cost Anomaly Detection** | 有効化（無料・週次レポート可） |

## 「料金を語るときの注意」

採用面接やクライアント提案でコストを語る際、誤りやすいポイント：

- **「Lambda は無料」は嘘** → 無料枠は月 100 万 requests + 40 万 GB-sec。**CloudWatch Logs の課金が現実のベース**
- **「CloudFront Invalidation は requests 単位」は誤解** → 正しくは **paths 単位**。`/*` 1 つの wildcard でも 1 path カウント
- **「SES は固定」は不正確** → 送信通数 / バウンス対応 / データ転送で変動。月 100 通なら数円、月 1 万通でも 100 円程度
- **「ACM は無料」は限定的** → パブリック証明書は無料だが、Private CA は別料金体系

これらを混同しない説明ができることが、AWS 課金体系を理解している証拠になります。
