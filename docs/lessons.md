## D. 技術的敗北（思ってたほど上手くいかなかったもの）

> 「合理的な意思決定」だけ並べると嘘くさくなる。**選んだけど苦戦した / 選び直そうか迷っているもの** を書く。

- **Next.js が想定より重かった** — 個人ブログ用途には大袈裟。`next build` が記事 50 件で 90 秒近くかかる。Astro / Hugo に移行を検討中だが、SSR 拡張の余地を残したくて据え置き。**移行できていないこと自体が苦い結果**
- **DynamoDB の書き味が辛い** — JOIN がなく、ローカル検索のたびに「Scan するか、GSI を 1 つ足すか」の判断を強いられる。問い合わせ程度のデータ量なら問題ないが、**データモデルが育つことを織り込んでいなかった** ことに後悔。代替案として S3 + Athena も検討中
- **Terraform module 設計で詰まった** — 初期に root 直書き → モジュール化 → 細分化 → 統合と 3 回書き直した。最終解の 5 モジュールも「本当に良いか」自信がない。**設計の迷いを残したまま運用に入っている** のが正直なところ
- **CloudWatch Logs Insights の学習コスト** — クエリ言語が独自で、本番障害の最中に思い出せず手間取ったことがある。Athena / Logs Live Tail / `aws logs tail` の使い分けがまだ曖昧
- **Lambda のローカル実行環境** — `sam local` / `serverless-offline` / `vitest` でモック を試したが、本番との挙動差で 2 回ハマった。**ローカルでは通るのに本番で失敗する** 系のデバッグに弱い
- **ACM 証明書の自動検証** — Terraform で投げると検証中の Pending タイムアウトに何度かハマった。手動で進めた方が早かった、というのは負け感のある気付き

## E. 妥協・後回し判断

> 全部の判断に正しい理由があるわけではない。「とりあえずこうしてる」を正直に。

- **WAF を入れていない** — 「平常時は過剰投資」と書いているが、本音は **「設定の手間が面倒で後回し」**。攻撃を受けたら入れる、で済ませている
- **Synthetics Canary を導入していない** — Runbook には書いたが、月数百円の追加コストとセットアップ時間で重い腰が上がっていない
- **Multi-region は構想止まり** — そもそも個人サイトに必要かどうか踏ん切りがついていない
- **コードカバレッジの目標値を決めていない** — Lambda の Jest テストはあるが、「カバレッジ 80%」みたいなルールは敷いていない。**「動くコード優先」で運用中**
- **依存パッケージの定期更新が追いついていない** — Renovate / Dependabot は入れているが、PR が溜まりがち。月初に一気に潰す運用

## F. いま悩んでいること（Open Questions）

> 「完成された設計」ではなく「考え続けている過程」を見せる場所。

### 構成判断の悩み

- **WAF の導入閾値** — 平常時は過剰投資、攻撃を受けてからでは遅い。中間解として「アラートベースで動的有効化」を Lambda + EventBridge で組めないか **構想中**（未実装）
- **マルチリージョン化のコスパ** — 個人サイトに ap-northeast-1 と us-west-2 の active-active は過剰だが、SES だけは冗長化したい
- **CMS 移行のタイミング** — 今は MDX で書いているが、外部寄稿者が入る時 microCMS に切り替えるか自前 admin を作るか未決
- **画像最適化のビルド時間** — 記事数が増えるとビルドが線形に遅くなる。Lambda で都度生成へ移すか、Cloudflare Images に逃すか未決

### コストと可観測性のトレードオフ

- **CloudWatch Logs Retention** — 短くするとデバッグ材料が減り、長くするとコストが膨らむ。今は 30 日だが、半年運用後に最適点を再評価予定

### IaC の悩み

- **module 粒度** — 細かく切ると apply の依存解決が遅い。粗いと変更影響範囲が広がる。**5 モジュール構成** が現状の答えだが、自信なし
- **dev / prod 分離** — 別バケット・別ステートで物理分離しているが、common module の更新時の整合性確認が手作業

## G. 思考プロセスの公開

これらの「迷い」自体を月次でブログに書き、半年後に **「あの時こう判断したが、結果はこうだった」** という形で結論を残していきます。

完成された答えではなく、**考え続けている過程を見せる** ことが、テンプレ的なポートフォリオとの最大の差別化要因だと考えています。
