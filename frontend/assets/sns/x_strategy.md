# X (Twitter) 運用方針 — iigtn

## 投稿頻度
- 平日: 1〜2 件 / 日。技術ノートは週 2〜3 件目安。
- 週末: 学習メモ・読了感想・運用ふりかえり中心。
- スレッド形式は「設計判断の比較」「ADR 抜粋」など、ブログにする前のメモ用途で使う。

## 投稿カテゴリ
1. **技術ノート** — AWS / Terraform / Linux で詰まったこと・解決したこと。
2. **iigtn.com 更新通知** — 新記事 / runbook 追加 / アーキ図変更などの差分共有。
3. **設計判断・選定理由** — なぜそれを選び、なぜ選ばなかったか。ADR 形式の短縮版。
4. **失敗ログ / Post-mortem** — 自分のミスを残す。信用は再現可能性から生まれる。
5. **OSS / 公式アップデート所感** — AWS What's New、Terraform リリース等を素直な一次反応で。

## やらないこと
- 炎上・政治・他者批判への参加。
- 引用 RT で他者の揚げ足取り。
- 案件の愚痴・クライアント特定可能な投稿。
- スクショの無断転載（公式 OG 画像 / 著者公開 OK のものに限る）。
- 数字を盛らない（フォロワー / 案件 / 単価で見栄を張らない）。

## 導線設計
- bio: iigtn.com の URL を明記。
- 固定ツイート: lab.iigtn.com への誘導 + GitHub `iigtn` org の案内。
- 技術記事は Zenn 開設後にダブル投稿（Zenn 本文 → X はサマリ + リンク）。
- GitHub での新リポジトリ公開時は X で告知して導線確保。

## 運用上の判断基準
- 「3 ヶ月後の自分が読み返して恥ずかしくないか」を 1 投稿ずつ自問。
- フォロワー数より、信頼できる相互フォロワーを 30 人作ることを優先。
- DM は基本開放、業務連絡は iigtn.com の問い合わせフォームに誘導。
