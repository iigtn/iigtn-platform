autoST — 自動売買リサーチ＆運用フレームワーク

autoST は、日本株 / 米国 ETF を対象に自動売買戦略を検証・運用する個人研究プロジェクトです。 Lambda + EventBridge + DynamoDB + S3 を Terraform で IaC 化し、ローカル PC で kabu STATION API と連携。20 を超える戦略を AI 共同レビューでバックテストし、最終的に 1545（米国 S&P500 ETF） + 200 日 MA + 月次キャッシュ確定型に収束しました。コードは内部運用のため非公開です。

A. 戦略の絞り込みプロセス（PDCA × AI 共同レビュー）

20 を超える候補戦略をバックテストし、AI 共同レビュー（ChatGPT との往復ディスカッション）で 弱点と過学習を指摘してもらいながら絞り込みました。

  1. v1〜v5: 日本株のスキャル・スイング戦略（VWAP 平均回帰 / RSI 二峰 / 上方修正サプライズ等）。手数料・スリッページ込みで現実的なリターンが取れる戦略を探索。
  2. v6〜v10: morning_trader / event_pivot / 半自動運用への振替。kabu STATION の制約（取引時間・手数料体系）に合わせた最適化。
  3. v0.1〜v0.6 (スキャル PDCA): VWAP / 出来高ブレイクの組合せ。スリッページ + 手数料込みで全パターン赤字 → スキャル戦略は廃止判断。
  4. v10〜v11: スイング 20 日高値ブレイク + イベント転換（上方修正の需給歪み）+ AI 判断層。月勝率と DD のトレードオフを再評価。
  5. 最終: 米国 ETF (1545) + 200 日 MA で月初 Buy / 月末 Sell の 月次 BuyHold に収束。複雑さを増やしても勝率が上がらないことが実証されたため。

「複雑さ ≠ 強さ」を実証したのが最大の学び。25 銘柄 + intraday 動的 exit のような高度な戦略を 5 周回しても、 単一 ETF + 200 日 MA + 月初/月末売買の方が長期成績は良かった。

B. 自動運転フロー（AWS サーバレス）

  1. EventBridge ― JST 平日朝 8:45（米国時間夜 19:45）に発火 → 日次の前場前準備 Lambda を起動。
  2. Lambda (data_collector) ― Yahoo Finance や J-Quants API から株価・出来高・市場ステータスを取得 → S3 (autoST-cache) に Parquet で保存。
  3. Lambda (strategy_runner) ― S3 から最新データを読み、200 日 MA / シグナル / ポジション状態を計算 → DynamoDB (autoST-positions) に書き戻し。
  4. Lambda (paper_trader) ― 仮想ポートフォリオで売買シミュ → 累積損益をメール（SES）で配信。
  5. ローカル PC (kabu STATION) ― 実発注は kabu STATION の REST API 経由でローカル PC から実行（PC を起動しておく必要あり）。Lambda → ローカルへの通知は SQS / SNS Email を経由。

C. IaC — Terraform 全部

AWS リソースは Terraform で管理。state は専用 S3 バケットに DynamoDB Lock 付きで保管。アカウントは 個人 AWS 環境（伏字）。

  • main.tf — provider、共通設定、tags
  • lambda.tf — Lambda Functions（data_collector / strategy_runner / paper_trader / event_watcher）+ Layer + 環境変数
  • iam.tf — Lambda 実行 Role + 最小権限 Policy（DDB / S3 / SES / Secrets Manager のみ）
  • dynamodb.tf — positions + signals + backtest_results テーブル（On-Demand）
  • s3.tf — Parquet キャッシュ・バックテスト結果バケット（Versioning + SSE-S3）
  • eventbridge.tf — 日次/週次/月次のスケジューラ Rule

D. リスク対策

  • auto_guard: VIX > 30 等のリスクオフ局面で全ポジションを自動クローズ → キャッシュ退避
  • regime_filter: 市場レジーム（強気・弱気・横ばい）に応じてポジションサイズを動的調整
  • stress_scenarios: コロナショック / リーマン / 1987 ブラックマンデー級の DD を再現する週次 stress test を CI で自動実行
  • テスト: 36 ケース pass、ChatGPT 共同レビューで Round 3 条件付 GO 判定

IaCTerraformstate を S3 backend に置き、複数環境（dev / prod）で再現性確保
レビューChatGPT（gpt-4o）共同 PDCA戦略の盲点・過学習を指摘してもらい、人間 1 人では出せない速度で改善

運用上の工夫

  • 戦略のバージョニング: v0.1〜v11 までを git タグ + S3 prefix で物理分離。再現性のためのスナップショット。
  • バックテストの自動 PDCA: ChatGPT に実行ログ + 結果を送り、改善案 → 実装 → 再実行を自動化。
  • WFO（Walk Forward Optimization）: 過学習回避のため、過去 X 年で学習 → 直近 1 年で検証する rolling window 評価を必須化。
  • コスト監視: Lambda 起動回数 + DDB 書き込みを CloudWatch Alarm で監視、想定超過時は SNS 通知。
